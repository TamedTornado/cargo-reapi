For both populations, every target was empty at start and every member started
before any member completed. There were no correctness violations. Executable
path relocation in the immediately preceding Linux measurement took 392 ms for
the large test snapshot after revision `ac1275a` replaced repeated 20 GB scans
with manifest-bound, validated offset writes.

The ten-member population is intentionally not listed as a result: it did not
pass. Ten portable ext4 target copies filled the server volume, causing the
independent `strace` observers and gate processes to fail with `ENOSPC`. That
failure identifies the remaining Linux scalability requirement: use a proven
reflink-capable cache/worktree filesystem or another isolation mechanism that
does not physically duplicate the complete linked target for every consumer.
